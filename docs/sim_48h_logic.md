# Логика сценария E — `_sim_48h()`

> Файл: `4_analytics.py`, строка 452

## Параметры

| Параметр     | Тип           | Описание                                      |
|-------------|---------------|-----------------------------------------------|
| `df`        | DataFrame     | 1-минутные свечи (high, low, close)           |
| `side`      | str           | Направление: `"BUY"` или `"SELL"`             |
| `entry`     | float         | Цена входа в позицию                          |
| `tp_prices` | list[float]   | Уровни тейк-профитов (TP1, TP2, TP3)         |
| `sl_orig`   | float         | Оригинальный стоп-лосс (не используется напрямую — SL динамический) |
| `total_usdt`| float         | Размер позиции в USDT                         |

Константы: `LEVERAGE = 20`, `TP_WEIGHTS = [0.5, 0.3, 0.2]`, `MAX_48H_CANDLES = 2880`.

---

## 1. Инициализация

- Веса TP нормализуются под количество уровней через `_norm_weights()`.
- Рассчитывается **порог ликвидации**: `entry ± (total_usdt / qty)`, где `qty = (total_usdt × 20) / entry`. Для BUY ликвидация ниже entry, для SELL — выше.
- Исход по умолчанию: `"E_timeout_48h"`.

## 2. Основной цикл (до 2880 свечей = 48 часов)

На каждой минутной свече последовательно проверяются:
1. Ликвидация
2. Тейк-профиты
3. Полное закрытие по TP
4. Стоп-лосс (динамический или троллинг)
5. Временные правила (36ч / 42ч)

## 3. Ликвидация

- Если цена (low для BUY, high для SELL) пробивает порог ликвидации — **запоминаем** номер свечи, но **не прерываем** цикл.
- Причина: нужно проверить, не сработал ли TP раньше ликвидации на той же свече.
- В финале: если ликвидация была **И** исход не полный TP **И** исход SL-подобный → PnL = −100% маржи, исход = `"E_liquidation"`.

## 4. Тейк-профиты и троллинг-стоп

- TP снимаются **последовательно** (TP1 → TP2 → TP3). За одну свечу могут сработать несколько.
- При каждом TP фиксируется прибыль на долю позиции (50% / 30% / 20%).
- **Троллинг-стоп** (подтягивание SL после TP):
  - После TP1 → SL на entry (безубыток).
  - После TP2 → SL на уровень TP1.
- Если все TP сработали (`remaining_w ≈ 0`): исход `"TPn"`, выход.

## 5. Динамический SL (до первого TP)

Пока `sl_locked = False`, SL зависит от текущего ROI:

| ROI            | SL                    | Логика                                |
|----------------|-----------------------|---------------------------------------|
| < −40%         | **None** (снят)       | Глубокая просадка — ждём таймаут 48ч  |
| −40% … −30%    | entry ± 41%           | Очень широкий стоп                    |
| −30% … −20%    | entry ± 31%           | Средний стоп                          |
| ≥ −20%         | entry ± 21%           | Стандартный стоп                      |

Знак: BUY → минус, SELL → плюс.

## 6. Срабатывание SL

- Если `active_sl` задан и свеча его пробивает — вся оставшаяся доля закрывается по цене SL.
- Исход: `"E_SL"` (без TP) или `"E_SL_after_TPn"` (после частичного профита).

## 7. Временные правила

### 36ч (≥ 2160 свечей)
- ROI 1%–1.5% → закрываем слабый профит. Исход: `"E_close_36_48h"`.
- ROI ≤ −10% → закрываем сильный убыток. Исход: `"E_close_minus10pct"`.
- ROI > 1.7% → держим дальше.

### 42ч (≥ 2520 свечей)
- ROI от −15% до −10% → закрываем по расчётной цене `entry ± 1%`. Исход: `"E_close_42h_minus1pct"`.

### 48ч (2880 свечей)
- Принудительное закрытие по последней close. Исход: `"E_timeout_48h"`.

## 8. Исходы

| Исход                    | Значение                                                        |
|--------------------------|-----------------------------------------------------------------|
| `TP1` / `TP2` / `TP3`   | Все TP сработали, позиция закрыта с прибылью                    |
| `E_SL`                   | Стоп-лосс без единого TP                                       |
| `E_SL_after_TP1/TP2`    | Стоп после частичного профита                                   |
| `E_close_36_48h`        | Закрытие слабого профита (1–1.5%) после 36ч                     |
| `E_close_minus10pct`    | Аварийное закрытие при убытке ≥10% после 36ч                    |
| `E_close_42h_minus1pct` | Закрытие умеренного убытка (10–15%) после 42ч, цена entry ± 1%  |
| `E_timeout_48h`         | Таймаут 48ч, закрытие по рынку                                  |
| `E_liquidation`         | Ликвидация, PnL = −100%                                        |

**Возвращаемый словарь:** `outcome`, `tps_hit` (0–3), `pnl` (USDT), `roi` (%), `candles`, `close_price`, `liquidation_at_candle`.
